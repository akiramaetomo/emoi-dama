# えもい玉 / emoi-dama

[English README](README.md)

**小さく心が動いた瞬間を、ころんと残す。**<br>
**見返したその瞬間は、美しく輝く。**

`えもい玉` は、日々のささやかな「えもいゴト」に手触り感を与える、
日常使いできるカレンダー＆メモアプリです。

食べ物がおいしかった。空がきれいだった。親の声が元気そうだった。
一日の終わりに少しほっとした。長く説明するほどではないけれど、
確かに心が動いた。そんな瞬間を、色のついた玉として手元に残します。

アプリをいつも触りたくなるよう、アプリそのものの「エモさ」も大事にしています。

公開プロトタイプ:

```text
https://akiramaetomo.github.io/emoi-dama/
```

現在の正式版は **0.8.0** です。詳しい更新履歴は
[CHANGELOG.md](CHANGELOG.md) を参照してください。

## 玉として残す

玉は、小さなお守りのようなものです。単なる記録の道具ではありません。
見て、触って、あとで眺めて、ときには大切な人にそっと預けることができます。

出来事の中身は、自分の中にしまっておく。けれど、
「何かが確かに起きた」という存在感は、玉、色、動き、音として感じられる。
そして紙のような `お預け状` / `預かり証` として外にも置ける。
この分離が、えもい玉の心理的な安全のキモです。

## 預ける / 預かる

えもい玉は、人に預けてもらったり、預かったりができます。
「預かり」はこのアプリの独自の考え方です。

感情は、それを持った人と切り離されないと考えます。だから、玉はあげたり、
配ったり、投げ銭のようにばらまいたりするものではありません。
あくまで、誰かの心の動きを、そっと預かるというテイです。

## 何に使うか

えもい玉を使うことで、今まで発生した「エモい感」を振り返られます。
SNS に投稿するほどでもない。人に言うほどでもない。けれど、なんとなく
残しておきたい。プチ笑い、プチ感動、プチ安心、プチ疲れ。
そういう些細なことを大事にします。

あとで見返して、「自分、何言ってるんだ？」と突っ込みたいときは、
中を開いてみましょう。その瞬間もまた、少しだけエモいかもしれません。

えもい玉は、幸せ達成アプリでも、感情発散ツールでも、ポジティブ／ネガティブ投稿の
SNS でもありません。日記共有でもありません。宗教とも違います。

ランキング、いいね、フォロワー、連続記録の圧力、承認のために感情を
整える感覚からは距離を置きます。多くの玉を集めることより、
ひとつの玉に意味があることを重視します。

## 現在のプロトタイプ

現在のプロトタイプは、ブラウザだけで動きます。

- 感情の小さな出来事を玉として作り、保存する
- Pixi/WebGL と Rapier で動く玉を転がす、つかむ、弾く、音を鳴らす。
  玉が多いときも端末に合わせて表示を調整する
- 保存済みの玉データを書き換えず、セッション中だけ使える `術` で、
  固定重力、疑似浮力、親玉、分割、再結合を試す
- 設定で通常時と術使用時の物理プロファイルを別々に調整する
- 玉の中身を見たり編集したりする
- 共通のコントロールバーから、Play、カレンダー、1日分の玉リストを
  行き来し、表示期間を切り替えて記憶を見返す
- `お預け状` / `預かり証` をQRや共有用画像として表示する。
  降臨GPSは、設定で明示的に有効にしない限り含めない
- サーバ DB なしで、受け渡し用QRに入った1つの玉のパケットを受け取る
- 手動バックアップや確認のために JSON をエクスポート / インポートする

プロトタイプ利用時の注意:

- データは現在のブラウザのローカルストレージにだけ保存されます。
- サーバ DB、アカウント、アナリティクス、広告、公開タイムラインはありません。
- 初期テスト中は、機微な個人情報を入力しないでください。
- URL と JSON 機能は試作段階の受け渡し手段であり、恒久的なバックアップ
  サービスではありません。

## まず触る

1. アプリを開きます。
2. 何かエモいと感じたら「＋」マークで玉を1つ作ります。
3. 軽く弾いたりして、動きと音を試します。固定重力、疑似浮力、親玉、
   セッション中だけの分割を試すときは `術` を開きます。
4. 中身を見たいときは、玉をタップします。
5. カレンダーや1日分の玉リストで見返し、Playの期間操作で表示範囲を
   切り替えます。
6. 人に玉を渡したいときは `玉を送る` を開き、相手にQRを見せるか、
   `画像で送る` を使います。降臨GPSを含める場合だけ、設定で明示的に
   有効にします。
